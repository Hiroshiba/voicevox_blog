# 日誌作成

実装作業を振り返り、日誌ファイルを作成する。

## 処理フロー

1. `date +%Y-%m-%d_%H%M%S` で現在日時を取得する
2. `diary/*.md` を検索し、直近の日誌があれば読んで記述スタイルを参考にする
3. `diary/YYYY-MM-DD_HHmmss.md` に日誌を書く
4. 呼び出し元に「DIARY_DONE」と報告する

## 記載内容

以下の3項目を書く。箇条書きを基本とし、具体的な事実を書く。

### レビュー内容

今回のレビューで指摘された内容と、それぞれの対応結果を記載する。レビュー指摘がなかった場合はその旨を書く。

### 見逃しの考察

レビューで初めて気づいた問題、またはレビューなしでは見逃していた可能性のある点を記載する。なければ「なし」と書く。

### 手こずったこと

実装中に詰まった点、想定外の問題、解決に時間がかかったことを記載する。なければ「なし」と書く。
