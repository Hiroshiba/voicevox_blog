このリポジトリは bulma フレームワークに依存しています。
その依存をなくし、tailwindcssとzag.jsを使用するように置き換えることが最終目標です。
zag.jsはReact版の他にAstroのアイランドを介さないバニラでも利用します。

タスクを順に進めていきましょう。

git add・git commit・git reset・git restoreは絶対に勝手に行わないでください。

コードを書き足す際は周囲にあるコードや兄弟ファイルのコードをよく観察して、フォーマット、書き方、処理順序、ファイル名など、なるべく既存のコードと同じスタイルで書いてください。

実行チェックには`pnpm start`で開発サーバーを立ち上げて、playwright-browser-operatorサブエージェントを使ってplaywright mcpを使った動作確認をしてください。
初回アクセス時は時間がかかることがありますが、ホットリロードが効くので、２回目以降の確認は速いはずです。

もしスクリーンショットを更新する際は`pnpm run test-build`を実行したあと`pnpm run test:e2e --update-snapshots`を実行してください。
`test-build`を忘れるとスクリーンショットが正しく更新されません。
変化しないはずのスクリーンショットが変わってしまった場合は、原因を特定して修正してください。

置き換えるときは既存のコンポーネントなどを直接Tailwind等で置き換えてください。
一時的に本体に問題があるかもしれませんが、その状態で進めてください。

tailwind化するPlanを練るとき、frontend-designのスキルを**必ず最初に**実行し、視点を参考にしてください。
あなたは必ずといって良いほどこのスキル実行を忘れるので、よく注意してください。
Planモードでも最初に実行し、実装モードでも最初に実行するようにしてください。
以前のBulmaのときのスクリーンショットも細かいところまで見てください。
目的はデザインを大きく変えずにtailwind化することなのでそこは注意してください。

primary color以外はtailwindにある色を使うようにしてください。

スクリーンショットを更新する際は、更新前にユーザーに対して開発サーバーで確認を行うようにaskskillで質問してください。
